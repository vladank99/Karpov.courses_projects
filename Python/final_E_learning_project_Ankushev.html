Задание №1. A/B–тестирование¶


Условие:

  • Одной из основных задач аналитика в нашей команде является корректное проведение экспериментов. Для этого мы применяем метод A/B–тестирования.
  • В ходе тестирования одной гипотезы целевой группе была предложена новая механика оплаты услуг на сайте, у контрольной группы оставалась базовая механика.
  • В качестве задания Вам необходимо проанализировать итоги эксперимента и сделать вывод, стоит ли запускать новую механику оплаты на всех пользователей.

Описание дата-сетов

  • groups.csv – файл с информацией о принадлежности пользователя к контрольной или экспериментальной группе
    (А – контроль, B – целевая группа)
  • group_add.csv – дополнительный файл с пользователями, который вам прислали спустя 2 дня после передачи данных
  • active_studs.csv – файл с информацией о пользователях, которые зашли на платформу в дни проведения эксперимента
  • checks.csv – файл с информацией об оплатах пользователей в дни проведения эксперимента.

Вопросы

  1. На какие метрики Вы смотрите в ходе анализа и почему?
  1. Имеются ли различия в показателях и с чем они могут быть связаны?
  1. Являются ли эти различия статистически значимыми?
  1. Стоит ли запускать новую механику на всех пользователей?

Логика решения¶

Условия:

  • Тестим новую механику оплаты услуг
  • Продукт – сайт

Мы имеем в данных:

  • айди пользователей
  • принадлежность к группе
  • доп. пользователей
  • айди пользователей, кто зашел на продукт
  • айди пользователей, кто купил продукт
  • доход с одного пользователя

Возникают следующие метрики на уме:

  • Конверсия в покупку (без учета времени)
  • Средний чек (с платящего пользователя)
  • ARPU (без учета времени)
  • ARPPU (без учета времени)

Мои условия¶

  • Мы не обговаривали с заказчиком метрики, поэтому не знаем, что считать выбросом
    • Следовательно, я считаю, что дата-сет не имеет выбросов
  • Договоримся, что мы работаем с конвенциональным значением 0.05 для p-value и 0.8 для мощности
  • Размеры эффекта в метриках также не были обговорены. Поэтому, решение на основе их значений я приму самостоятельно!

  1. Склеить основной дата-сет и group_add, с помощью pd.concat (чтобы данные склеились сверху-вниз)

Общий вывод по распределениям:¶

  • Каждая метрика (кроме конверсии) имеет распределение близкое к ненормальной форме
  • Однако, параметрические тесты могут это проигнорировать, согласно правилам ЦПТ (в частности т-тест)
  • С помощью бустрапа мы можем частично это продемонстрировать (распределения разниц между средними будут стремиться к нормальной форме). Следовательно, мы сможем использовать параметрические тесты (для оценки мощности)

# Создать бинарную колонку – status
# + Новый дата-сет active_users для таблицы сопряженности
active_users = full_data.query('activity == True')
active_users['status'] = active_users.rev > 0
active_users['status'] = active_users['status'].map({True:'pay', False:'active'})

# Таблица сопряженности: встречаемость
crosstab_table = pd.crosstab(active_users.grp, active_users.status)

# Таблица сопряженности: проценты по всему
crosstab_table_percentage = round(pd.crosstab(active_users.grp,
                                              active_users.status,
                                              normalize='all'),2)

# Хи-тест
c, p, dof, expected = stats.chi2_contingency(crosstab_table)
print('Chi_test_results: \n', 'chi_value:', c, '  df:', dof,
      '  p-value:', p, '\n \n',
      'H0: Взаимосвязи между Статусом и Группой нет \n',
      'H1: Взаимосвязь между Статусом и Группой присутствует \n')

if p < 0.05:
    print('p_value:',round(p,3),
          'принимаем альтернативную гипотезу H1: взаимосвязь между Статусом X Группа присутствует')
else:
    print('p_value:', round(p,3),
          'отклоняем альтернативную гипотезу H1: взаимосвязи между Статусом X Группа нет')

    FROM final_project_check
    GROUP BY st_id, subject

SELECT

    act.subject as a_subject,--предметы активных пользователей
    act.correct as correct, --правильные ответы активных пользователей
    partial.st_id as w_id, --id всех пользователей

def metrics(path:str, id_column:str, group_column:str):
    '''
    Функция подгружает дополнительный дата-сет, пересчитывает и строит графики по основным метрикам 
    
    :path – путь к дополнительному дата-сету (str)
    
    :id_column – название колонки id в дополнительном дата-сете (str)
    
    :products – название колонки grp в дополнительном дата-сете (str)
    '''    
    
    # Загрузка основных дата-сетов
    groups = pd.read_csv('groups.csv', sep=';')
    active_studs = pd.read_csv('active_studs.csv')
    checks = pd.read_csv('checks.csv', sep=';')

    # Исключение №1 о формате таблиц
    if '.csv' in path:
        group_add = pd.read_csv(path)
    else:
        print('Ошибка 1: напишите нужный формат таблицы: .csv')
    
    # Меняем названия колонок (если прислали отличные от исходных)
    group_add.rename(columns = {id_column:'id', group_column:'grp'}, 
                                inplace = True)

    # ––––Склеивание––––––
    # Склеивание основного дата-сета и дополнительного
    connected_data = pd.concat([groups, group_add],
                               ignore_index=True, sort=False)

    # Добавление булевой колонки
    connected_data['activity'] = np.isin(connected_data['id'],
                                         active_studs['student_id']).tolist()

    # Переименование колонки, чтобы merge сработало
    checks = checks.rename(columns={'student_id': 'id'})

    # Склеивание с помощью left-merge
    full_data = connected_data.merge(checks, on='id', how='left')

    # Заполним NA в колонке с чеками
    full_data['rev'] = full_data.rev.fillna(0)

    # Изменение статус багованных пользователей
    full_data['activity'] = np.where(full_data['rev'] > 0, True,
                                     full_data.activity)

    # ––––Подсчет конверсии––––––
    cr_data = full_data.query('activity == True') \
                       .groupby('grp', as_index=False) \
                       .agg({'activity':'count'})

    cr_data['payment'] = full_data.query('rev > 0') \
                                  .groupby('grp').agg({'activity':'count'}) \
                                  .activity.to_list()

    cr_data['cr_coeff'] = round((cr_data.payment / cr_data.activity) * 100, 2)

    # ––––Подсчет среднего и медианного чека––––
    check_data = full_data.query('rev > 0').groupby('grp', as_index=False) \
            .agg({'rev':'mean'}).round(2).rename(columns={'rev':'avg_check'})

    check_data['median_check'] = full_data.query('rev > 0') \
            .groupby('grp', as_index=False) \
            .agg({'rev':'median'}).round(2).rev.to_list()

    # ––––Подсчет ARPU и ARPPU––––
    arpu_data = full_data.query('activity == True') \
            .groupby('grp', as_index = False).agg({'id':'count', 'rev':'sum'}) \
            .rename(columns={'id':'num_users'}).round()

    arpu_data['arpu'] = round(arpu_data['rev'] / arpu_data['num_users'], 1)

    arppu_data = full_data.query('activity == True & rev > 0') \
            .groupby('grp', as_index=False).agg({'id':'count', 'rev':'sum'}) \
            .rename(columns={'id':'num_users'}).round()

    arppu_data['arppu'] = round(arppu_data['rev'] / arppu_data['num_users'], 1)

    # ––––Графики––––

    # Создаем две группы
    pay_user_A = full_data.query('rev > 0 & grp == "A"')
    pay_user_B = full_data.query('rev > 0 & grp == "B"')

    # Создаем 3 рамки для графиков
    fig, axes = plt.subplots(4, 2, figsize=(15, 5))

    # Общее название
    fig.suptitle('РАСПРЕДЕЛЕНИЕ МЕТРИК')

    # Названия для отдельных графиков
    axes[0,0].set_title('Кол-во активных и платящих пользователей')
    axes[0,1].set_title('Конверсия по группам')

    axes[1,0].set_title('Распределение чеков группы A')
    axes[1,1].set_title('Распределение чеков группы B')

    axes[2,0].set_title('Кол-во пользователей и доход (ARPU)')
    axes[2,1].set_title('ARPU между группами')

    axes[3,0].set_title('Кол-во пользователей и доход (ARPPU)')
    axes[3,1].set_title('ARPPU между группами')

    # Размер графиков
    fig.set_size_inches(20, 30)

    # Первый график
    # Перевод с широкого формата на длинный 
    cr_data_long = cr_data.rename(columns={'activity':'activity_status', 
                                           'payment':'payment_status'}) 

    cr_data_long = pd.melt(cr_data_long, id_vars='grp', 
                           value_vars=['activity_status', 'payment_status'])

    # Рисуем график №1
    graph1 = sns.barplot(x="variable", y="value", hue="grp", 
                         data=cr_data_long, ax=axes[0,0])

    # Подписи для первого графика (цифры над столбцами)
    for container in graph1.containers:
        graph1.bar_label(container)

    # Рисуем график №2
    graph2 = sns.barplot(x="grp", y="cr_coeff", 
                         data=cr_data, ax=axes[0,1])

    # Подписи для второго графика (цифры над столбцами)
    for container in graph2.containers:
        graph2.bar_label(container)

    # Третий график
    graph3 = sns.distplot(pay_user_A.rev, color="g",ax=axes[1,0]) 
    graph3.axvline(x = pay_user_A.rev.mean(), color='red')
    graph3.axvline(x = pay_user_A.rev.median(), color='blue') 

    # Четвертый график
    graph4 = sns.distplot(pay_user_B.rev, ax=axes[1,1])
    graph4.axvline(x = pay_user_B.rev.mean(), color='red')
    graph4.axvline(x = pay_user_B.rev.median(), color='blue')

    # Легенды
    bbox_props = dict(boxstyle="round", fc="w", ec="0.5")

    # Легенда среднего для третьего графика
    axes[1,0].annotate(f"Mean = {round(pay_user_A.rev.mean(),2)}",
                       xy=(0.98, 0.95), xycoords='axes fraction',
                       fontsize=16, horizontalalignment='right',
                       bbox=bbox_props, color='red')

    # Легенда медианы для третьего графика
    axes[1,0].annotate(f"Median = {round(pay_user_A.rev.median(),2)}",
                       xy=(0.98, 0.88), xycoords='axes fraction',
                       fontsize=16, horizontalalignment='right',

    # Легенда среднего для четвертого графика
    axes[1,1].annotate(f"Mean = {round(pay_user_B.rev.mean(),2)}",

    # Легенда медианы для четвертого графика
    axes[1,1].annotate(f"Median = {round(pay_user_B.rev.median(),2)}",

    # Пятый график
    # Перевод с широкого формата на длинный 
    arpu_data_long = pd.melt(arpu_data, id_vars='grp',

    # Рисуем график №5
    graph5 = sns.barplot(x="variable", y="value", hue="grp",

    for container in graph5.containers:
        graph5.bar_label(container)

    graph6 = sns.barplot(x="grp", y="arpu",
                         data=arpu_data, ax=axes[2,1])

    # Подписи для шестого графика (цифры над столбцами)

    # Перевод с широкого формата на длинный 
    arppu_data_long = pd.melt(arppu_data, id_vars='grp',

    # Рисуем график №7
    graph7 = sns.barplot(x="variable", y="value",
